Getting Started with CSS Custom Properties

By Alex Kim · March 15, 2026 · 5 min read

CSS custom properties — also called CSS variables — let you store values in reusable tokens. Unlike preprocessor variables (Sass, Less), they are truly part of the cascade: they can be scoped to elements, overridden by media queries, and even updated at runtime with JavaScript.

Defining Custom Properties

Custom properties are defined with a double-dash prefix and are typically placed on the :root pseudo-element so they're globally available:

:root {
  --color-accent: #4f46e5;
  --font-sans: system-ui, sans-serif;
  --space-4: 1rem;
}

You reference them with the var() function. You can also provide a fallback value as the second argument:

button {
  background-color: var(--color-accent, #4f46e5);
  font-family: var(--font-sans);
}

Scoping and Overriding

Because custom properties cascade like any other CSS property, you can scope them to a specific element. This makes theming straightforward — define a [data-theme="dark"] selector that overrides your root properties:

[data-theme="dark"] {
  --color-bg: #0f172a;
  --color-text: #f1f5f9;
}

Every element that uses var(--color-bg) will automatically pick up the new value. No class toggling cascade, no JavaScript swapping stylesheets.

Runtime Updates

One of the most powerful features is that you can update custom properties at runtime with JavaScript:

document.documentElement.style.setProperty('--color-accent', '#e11d48');

This makes dynamic theming — like letting users pick an accent color — trivial to implement.

Wrapping Up

CSS custom properties are one of the most impactful additions to the CSS specification in years. Start with a small set of tokens for color, typography, and spacing, and you'll find that your stylesheets become dramatically easier to maintain and modify.
